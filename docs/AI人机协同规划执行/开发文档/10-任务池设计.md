### AI 人机协同规划执行开发文档

## 十、任务池设计

### 任务粒度原则

每个任务都必须满足：

- 单一目标明确
- 输入和约束明确
- 成功条件明确
- 可独立追踪状态
- 可单独人工介入

### 推荐任务类型

- `analysis`
- `read`
- `modify`
- `verify`
- `checkpoint`
- `manual_review`
- `replan`

### 任务调度规则

首版采用：

- 按依赖关系解锁 `ready`
- 在 `ready` 中按优先级选择
- 同优先级按 `sequence` 执行
- 高风险任务先转 `waiting_human`
- 失败任务优先判断是否重试

### 任务状态流转

```mermaid
stateDiagram-v2
    [*] --> pending
    pending --> ready
    ready --> running
    running --> done
    running --> waiting_human
    running --> failed
    waiting_human --> ready
    waiting_human --> skipped
    failed --> ready
    failed --> cancelled
```

### 任务池调度循环

```mermaid
flowchart TD
    A[载入会话与 Execution Pack] --> B[扫描任务池]
    B --> C[选出 ready 任务]
    C --> D[Agent 领取一个任务]
    D --> E[执行任务]
    E --> F{结果}
    F -- 成功 --> G[回写 result 与 artifacts]
    G --> H[解锁依赖任务]
    H --> B
    F -- 高风险或需确认 --> I[状态置为 waiting_human]
    I --> J[人工处理]
    J --> B
    F -- 失败 --> K[重试/转人工/重新规划]
    K --> B
```
