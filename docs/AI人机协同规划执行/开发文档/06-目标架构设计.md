### AI 人机协同规划执行开发文档

## 六、目标架构设计

### 架构分层

#### 1. 交互层（Frontend）
负责：

- 输入任务目标
- 展示主方案树
- 选中节点/模块
- 展示节点执行思路
- 展示人工审查表单
- 展示 Execution Pack
- 展示任务池运行态
- 提供人工介入入口

#### 2. 协同规划层（Planner / Detail / Review）
负责：

- 根据任务目标生成主方案
- 根据选中节点生成详细执行思路
- 接收人工审查修改
- 合并生成最终可执行版本

#### 3. 执行编排层（Execution Pack / Task Pool / Scheduler）
负责：

- 将方案转换为 Execution Pack
- 将 Execution Pack 拆分为任务池
- 维护任务依赖、优先级和状态
- 为 Agent 选择下一个待执行任务
- 触发人工介入与重规划

#### 4. 执行层（Agent Executor）
负责：

- 每次消费一个 `ready` 任务
- 按任务约束执行工具调用
- 回写任务结果
- 标记风险与失败
- 返回下一步建议

#### 5. 状态层（DB + Cache + Bubble）
负责：

- 保存会话
- 保存主方案/节点思路/审查结果
- 保存任务池状态
- 保存人工介入记录
- 保存执行气泡流
- 支撑恢复与回放
